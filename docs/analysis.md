# Этап 1. Анализ предметной области

## Назначение системы

Информационная система складского учёта предназначена для автоматизации процессов
регистрации товаров, контроля их остатков и фиксации складских операций
(поступление, списание) в малых и средних организациях.

## Потенциальные пользователи

| Роль | Задачи |
|---|---|
| Кладовщик | Регистрация поступлений и списаний, просмотр остатков |
| Менеджер | Контроль запасов, формирование заявок на пополнение |
| Руководитель | Анализ статистики, выявление дефицита |

## Основные сущности

| Сущность | Атрибуты |
|---|---|
| Категория | id, название |
| Товар | id, название, артикул, категория, единица измерения, минимальный остаток |
| Остаток | id, товар, количество, местоположение |
| Операция | id, тип (поступление/списание), товар, количество, дата, ответственный, примечание |

## Актуальность

Ручной учёт на бумаге или в Excel не обеспечивает:
- Актуальность данных об остатках в реальном времени
- Историю операций с привязкой к ответственному
- Автоматического контроля минимальных запасов
- Удобного поиска и фильтрации
